# Глава 1: Потоки в сетях

Если раньше рёбра графа были дорогами, теперь они будут **трубами**.

Через трубы будем качать "поток" --- какой-нибудь материал (воду/нефть) или сетевой трафик.
Поток, текущей по трубе, можно измерить: например $5$ кубометров в секунду (сколько протекает через сечение в единицу времени) или $10$ гигабит в секунду.
Трубы не резиновые: у каждой трубы указана "пропускная способность" --- какой максимальной величины поток через неё разрешено пропускать.

Имея пропускные способности, уже можно гонять поток по трубам. Не хватает только цели.
Выделим две вершины: исток $s$ и сток $t$. Будем качать нефть из $s$ в $t$.
Задача: с какой максимальной скоростью можно поставлять нефть из $s$ в $t$ на заданной сети труб?

В теории "сеть" --- это граф с выделенными истоком $s$ и стоком $t$, а также с проставленными у рёбер пропускными способностями.
Вышеописанная задача --- это задача поиска максимального потока в сети $N$.

Заметим, что трубы могут быть разные:
1. Двусторонние: пропускная способность $c$ в обе стороны.
2. Односторонние: пропускная способность $c$ в одну сторону и $0$ в обратную.

## Хранение сети в памяти

### Матрица смежности.

* $c(u, v)$ --- пропускная способность трубы из $u$ в $v$ ($0$ если трубы нет);
* $f(u, v)$ --- величина потока по трубе из $u$ в $v$.

Можно хранить эти величины в матрицах.

Такое представление плохо работает на разреженных графах.
Зато оно удобно для теории: далее оно будет использоваться в лекции.
Кроме того, большая часть задач из набора решается таким образом.

### Списки инцидентности.

Можно хранить для каждой вершины список рёбер, которые из неё выходят.

**Важно:** в задачах на поток **все** рёбра "двойные", у каждого ребра есть обратное!
Если по ребру $(u, v)$ течёт поток $f$, то по обратному ребру всегда течёт поток $-f$.
В алгоритмах иногда нужно находить для ребра обратное.

Простой способ хранить списки инцидентности:
```
struct edge_t {
    // Откуда/куда идёт это ребро (номера вершин).
    int from, to;

    // Пропускная способность этого ребра в этом направлении.
    int cap;

    // Величина потока по этому ребру в этом направлении.
    int flow;
};

// Классический список рёбер.
edge_t edges[/* ... */];

// Списки инцидентности.
int_list_t inc[/* ... */];
```

Здесь $inc[u]$ --- это список **номеров** (в $edges$) всех рёбер, которые выходят из $u$.

При построении графа, когда очередное ребро добавляется в граф, **сразу** после этого надо добавить обратное ребро.
Тогда в массиве рёбра $edges[2k]$ и $edges[2k+1]$ будут обратными друг другу (для всех $k$).
Если так сделать, то можно легко вычислить по номеру ребра $z$ номер обратного ребра $nz$.
Короткий способ с использованием побитового XOR: \mintinline{c}{nz = z ^ 1}.

Чтобы было удобно строить сеть, для добавления трубы лучше написать отдельную функцию,
которая добавит соответствующие ей элементы и в $egdes$, и в $inc$.

## Потоки и разрезы

Чтобы $f$ было потоком, должны выполняться условия:
1. $f(u,v) = -f(u,v)$         (кососимметричность).
      Существенно упрощающает работу с потоками.
2. $\sum\limits_v{f(u,v)} = 0$         (закон сохранения потока --- для всех $u$ за исключением $s$ и $t$).
      Иными словами: сколько в $u$ втекает, столько и вытекает (нефть не теряется).
      В силу кососимметричности втекающие потоки в сумме учитываются со знаком минус, а вытекающие --- со знаком плюс.
3. $f(u,v) \le c(u,v)$         (ограничение на пропускную способность).


Разрезом называется разбиение вершин графа на две части, так что исток оказывается в левой части, а сток --- в правой:
* $A \cup B = V$;
* $A \cap B = \emptyset$;
* $s \in A$;
* $t \in B$.

Разрез обозначается как: $A|B$.

Рёбра разреза --- это все рёбра, у которых начало находится в $A$, а конец --- в $B$ (т.е. $\{(u, v) \in E | u \in A, v \in B\}$).
Для разреза можно определить: поток через него, и его пропускную способность.
Для этого надо просто просуммировать соответствующую величину по всем рёбрам разреза:
* $f(A|B) = \sum\limits_{u \in A} \sum\limits_{v \in B} f(u, v)$;
* $c(A|B) = \sum\limits_{u \in A} \sum\limits_{v \in B} c(u, v)$.

Можно заметить, что:
1. $f(A|B) \le c(A|B)$ --- очевидно.
2. Число $f(A|B)$ одинаковое для любого разреза.
      Это очевидно из интуитивных соображений: если потёрь нет,
      через границу $A|B$ должно течь ровно столько потока, сколько вытекает из истока.

Величиной потока называется $f(A|B)$ для любого разреза --- т.е. с какой скоростью мы переправляем нефть из $s$ в $t$.
В частности, его можно посчитать как суммарный вытекающий из $s$ поток, или как суммарный втекающий в $t$ поток:
$$
    |f| = f(A|B)
    \begin{cases}
        A = \{s\},             & B = V \setminus \{s\};\\
        A = V \setminus \{t\}, & B = \{t\}.
    \end{cases}
$$

Классические задачи на потоки:
1. Найти максимальный поток, то есть поток $f$, для которого $|f| = \max$.
      Фактически показывает, как нужно качать нефть, чтобы получилась максимальная скорость.
2. Найти минимальный разрез, то есть разрез $A|B$, для которого $c(A|B) = \min$.
      Показывает, где в сети "узкое место", то есть в какой разрез следует добавлять трубы, чтобы увеличить максимальный поток.
      Также разрез возникает в задачах в стиле: удалить набор рёбер минимального суммарно веса,
      чтобы отделить вершины $s$ и $t$ друг от друга.

## Метод Форда-Фалкерсона

Назовём "остаточной сетью" для потока $f$ в сети $N$ такую сеть $N_f$, которая от исходной сети $N$ отличается данными изменениями:
1. У каждого ребра "остаточная" пропускная способность равна:
      $$c_f(u,v) = c(u,v) - f(u,v).$$
      Обычно считается, что ребро присутствует в остаточной сети только когда $c_f(u,v) > 0$ для него.
2. Поток $f$ в остаточной сети $N_f$ стирается.

**Важно:** даже если трубы были односторонними, то в остаточной сети появляются обратные рёбра!
Обратные рёбра необходимы алгоритмам поиска максимального потока, чтобы правильно работать.
Без этого они не могут "передумать" о своих решениях, т.е. уменьшить величину потока, текущего по некоторому ребру.

Остаточная сеть показывает, сколько можно дополнительно пропустить потока по каждой трубе в сети.
В частности, если в остаточной сети $N_f$ построить какой-то поток $\Delta f$, то его можно добавить к текущему потоку $f$ в исходной сети $N$.
Величина потока $|f|$ при этом увеличится в точности на величину потока $|\Delta f|$.

Можно брать поток $g$ простого вида, например поток вдоль пути $P$.
Любой путь из $s$ в $t$ в остаточной сети называется "дополняющим путём".
По нему можно пустить поток, величина которого определяется минимальной остаточной пропускной способностью вдоль пути.
Этот поток можно добавить к текущему потоку в исходной сети $N$, тем самым увеличив её.

Метод Форда-Фалкерсона поиска максимального потока:
* $f \gets 0$;
* пока в остаточной сети $N_f$ есть дополняющий путь $P$: добавляем поток вдоль пути $P$.

При добавлении потока имеет смысл определить минимальную остаточную пропускную способность вдоль пути.
Именно столько потока мы может добавить вдоль данного пути без дополнительных ухищрений.

Замечание: в алгоритмах остаточная сеть $N_f$ явно не строится: её рёбра определяются "на лету" вычитанием $c(u,v)-f(u,v)$.
При проталкивании (добавлении) потока вдоль пути нужно также обновить поток по обратным рёбрам: здесь пригодится умение находить обратное ребро.

Вопрос: а верно ли, что когда дополняющие пути закончатся, получится именно максимальный поток?
Может быть мы упрёмся в потолок где-то ниже?

## Теорема Форда-Фалкерсона.

Пусть $f$ --- поток в сети $N$. Тогда эти условия эквивалентны:
1. $|f| = \max$ (поток является максимальным).
2. В остаточной сети $N_f$ нет дополняющего пути из $s$ в $t$.
3. Существует разрез $A|B$, у которого пропускная способность равна $|f|$ (нетрудно заметить, что такой разрез минимальный).

В доказательстве теоремы видно, как искать минимальный разрез:
1. Строим максимальный поток $f$.
2. Смотрим на остаточную сеть $N_f$.
3. Запускаем в $N_f$ обход из $s$, все достижимые вершины кладём в $A$.
4. В $B$ кладём все остальные вершины.

Тогда $A|B$ --- минимальный разрез.

Конкретные алгоритмы на основе метода Форда-Фалкерсона.

1. На каждом шаге ищем **любой** дополняющий путь $P$, например используя $DFS$.
      Можно заметить, что если все пропускные способности целые, то и поток в алгоритме всегда целый.
      Значит на каждом шаге величина потока увеличивается хотя бы на $1$ --- независимо от выбора пути.
      Тогда всего путей будет не больше $|f|$.

      Учитывая, что $DFS$ работает за $O(E + V)$, получаем (игнорируя $O(V)$):
      Время работы: $O(|f| \cdot E)$
      Это хорошая оценка, если максимальный поток целый и **маленький**.

      Интересный факт: если пропускные способности иррациональные,
      то на специально подобранной сети и при специальном выборе дополняющих путей метод Форда-Фалкерсона может работать **вечно**.

2. На каждом шаге ищем минимальный по количеству рёбер дополняющий путь, используя $BFS$.
      Это называется "алгоритм Эдмонса-Карпа".
      Известно, что в этом случае количество дополняющих путей не превышает $O(V \cdot E)$
      (это крайне пессимистичная оценка: на большинстве примеров нужно гораздо меньше путей)
      Получается общая оценка времени: $O(V \cdot E^2)$.


Полный алгоритм поиска максимального потока:

```
f = 0
while true
    // {Часть 1: BFS.}
    mark[*] = false
    mark[s] = true
    Q.Enqueue(s)
    while not Q.IsEmpty()
        u = Q.Dequeue()
        for (u, v) in E
            // Проверяем cf(u, v) - остаточную пропускную способность.
            if c(u, v) - f(u, v) > 0
                if not mark[v]
                    mark[v] = true
                    Q.Push(v)
                    F[v] = u

    if not mark[t]
        // Нет дополняющего пути.
        break

    // Часть 2: определяем величину.
    CP = infinity
    x = t
    // Восстанавливаем путь P первый раз.
    while x != s
        y = F[x]
        CP = min(CP, c(y, x) - f(y, x))
        x = y

    // Часть 3: увеличиваем поток.
    x = t
    // Восстанавливаем путь P второй раз.
    while x != s
        y = F[x]
        f(y, x) = f(y, x) + CP
        // Не забываем поменять поток по обратному ребру!
        f(x, y) = f(x, y) - CP
        x = y
```
